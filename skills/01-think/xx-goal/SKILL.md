# 目标定义

> 在写第一行代码之前，先想清楚：为谁解决什么问题，怎么判断成功。

## 这个技能做什么

帮你在 20 分钟内完成从"我有个想法"到"这是我要验证的核心假设"的转化。输出一张**目标定义卡**。

大多数产品失败不是因为代码写得差，而是因为一开始就没想清楚：
- 给谁用？（不是"所有人"）
- 解决什么问题？（不是"做个工具"）
- 怎么判断成功？（不是"用户量"这种虚荣指标）

本技能面向"用自然语言提需求、看结果提优化"的用户：你用自然语言描述想法和目标用户，AI 负责按框架追问、提炼、检查虚荣指标；你只需判断"这个目标是不是我想要的、用户描述准不准"。

**AI 执行约束：** 每一步产出必须能让用户判断"对/不对"，不能只输出抽象结论。用户分层必须具体到职业 + 场景，北极星指标必须能覆盖 HEART 至少 2 个维度，否则标"未达标，需补"。

---

## 三步流程

### 第一步：用户分层（10 分钟）

**要回答：谁是最早期的用户？**

不是"所有需要 XX 的人"，而是**具体到能想象出一张脸的人群**。

**引导问题：**
1. 你的产品帮谁完成任务？（说出他们的职业、年龄、日常场景）
2. 这群人现在怎么解决这个问题的？（替代方案）
3. 为什么他们愿意用你的而不是现有的方案？（10 倍好 or 10 倍便宜 or 全新体验）
4. 谁是"早期采用者"？（最痛、最急、最愿意尝试不完美方案的那批人）

**AARRR 用户生命周期检查：**
```
获取(Acquisition)：这些人从哪里知道你的产品？
激活(Activation)：他们第一次用，什么时刻会"啊这个有用"？
留存(Retention)：什么会让他们第二天还来？
推荐(Referral)：他们会主动分享给谁？
收益(Revenue)：他们愿意为什么付费？
```

如果 5 个环节中任何一个想不清楚，这个方向就需要再斟酌。

**判断标准：** 用户描述必须具体到"能想象出一张脸"——能说出职业、日常场景、现在用什么替代方案。说"所有人"或"中小企业主"这种宽泛词，视为不达标。

**AI 执行约束：** 若用户描述用了"所有人""一般用户""年轻人"等宽泛词，必须追问"具体是哪一类人，在什么场景下用"，不能直接放过。

---

### 第二步：北极星指标（5 分钟）

**要回答：如果只能看一个数字，看什么？**

北极星指标（North Star Metric）是**最能反映你给用户创造了多少核心价值的单一指标**。

**好指标 vs 坏指标：**

| 坏指标（虚荣指标） | 好指标（北极星指标） |
|------------------|-------------------|
| 注册用户数 | 首次完成核心任务的用户比例 |
| 页面浏览量 | 每周成功达成目标的次数 |
| 下载数 | 7 天留存率 |
| 收入 | 单位用户创造的核心价值 |

**北极星指标的 4 个特征：**
1. **客户中心** — 衡量用户获得的价值，不是你的收入
2. **领先性** — 预示未来增长，不是只看过去
3. **持久性** — 产品演进过程中依然有意义
4. **可理解** — 团队每个人都能一句话解释

**HEART 框架辅助检查：**
```
Happiness（满意度）：用户满意吗？怎么测？
Engagement（参与度）：用户多频繁使用？
Adoption（采用率）：新用户开始用了吗？
Retention（留存率）：用户回来吗？
Task Success（任务成功）：用户能完成想做的事吗？
```

你的北极星指标应该能覆盖 HEART 中至少 2 个维度。

**判断标准：** 北极星指标必须是"用户获得的价值"，不是"你的收入/注册量"。一句话能解释清楚。能覆盖 HEART 至少 2 个维度。

**AI 执行约束：** 若指标是"注册数/下载量/收入/DAU"这类虚荣指标，必须标记"虚荣指标，需替换"并给出至少 2 个候选北极星指标供用户选。

---

### 第三步：假设验证设计（5 分钟）

**要回答：我最不确定的是什么？怎么用最小成本验证？**

把前两步的结论转化为**可证伪的假设**：

```
假设：[目标用户] 在 [场景] 下 需要 [完成任务]，
      但现有方案 [痛点]，
      如果我们提供 [解决方案]，
      他们会 [预期行为]，
      我们可以用 [指标] 验证。

风险最高的假设是：____________
验证方法：____________
验证成本：____________（时间/金钱）
验证标准：____________（什么数字算通过）
```

**验证方法梯度（从低成本到高成本）：**
1. **桌面验证**：搜索有没有类似产品、看竞品评价里抱怨什么
2. **人工验证**：找 5 个目标用户聊，不提你的方案，只聊他们现在怎么做的
3. **落地页验证**：做一个介绍页，放"立即体验"按钮，看点击率
4. **手工 MVP**：用微信群/Excel/人工服务模拟核心流程
5. **代码 MVP**：只做核心功能，砍掉一切非必要的东西

**优先验证最危险的假设**，不是最容易做的功能。

**判断标准：** 假设必须可证伪（能说出什么数据算通过、什么算失败）。验证方法必须从低成本梯度选起，前一个不通过就不做下一个。

**AI 执行约束：** 若"风险最高的假设"和"验证方法"对不上（比如最高风险是付费意愿，却选了桌面验证），必须指出并建议改用落地页或手工 MVP 验证付费意愿。

## 验收清单

目标定义是否可靠：

- [ ] 目标用户具体到职业 + 场景，没用"所有人"等宽泛词
- [ ] 核心问题是用户真痛的，不是脑补的功能需求
- [ ] 北极星指标是"用户价值"不是"虚荣指标"，一句话能解释
- [ ] 北极星指标覆盖 HEART 至少 2 个维度
- [ ] 最危险假设可证伪，有明确的通过/失败数字
- [ ] 验证方法与最危险假设匹配（付费意愿别用桌面验证）
- [ ] 验证方法从低成本梯度选起
- [ ] AARRR 5 个环节都能说清

**AI 执行约束：** 任意一项不满足，必须明确标注"该项证据不足/不达标"，不能补全脑补。

## 示例：小象取色 的目标定义卡

### 产品名称
小象取色 — 摄像头实时取色 + AI 中式命名 + 配色海报

### 目标用户
每周做配色方案 ≥3 次的平面/UI 设计师，常接国风、品牌类项目

### 核心问题
1. 取色后只有 Hex，交付客户看不懂
2. 中式色名难查，要翻官网扒，一个色找半天
3. 有了色名也凑不出整套配色方案

现有替代方案：搜官网查色名 / 自己编名字 / 直接发 Hex

### 独特价值
摄像头一秒取色，AI 一秒出中式色名 + 整套配色方案，还能生成海报直接交付

### 北极星指标
**每周成功"取色→命名→生成配色"的完整闭环次数**（覆盖 Task Success + Engagement）

输入指标：
- 取色按钮点击率（激活）
- 命名结果采用率（任务成功）
- 配色方案生成率（参与度）
- 海报生成/分享率（推荐前兆）
- 7 天回访率（留存）

### 早期采用者
在小红书发配色笔记的国风设计师、接品牌 VI 项目的独立设计师

### 最危险假设
设计师愿意把 AI 命名结果直接交付给客户（而不是只当参考）
验证方法：手工 MVP（微信群，人工帮 10 个设计师命名 + 配色，看几人会直接用于交付）
验证标准：≥4/10 人把结果直接用于客户交付，且 ≥3 人问"什么时候上线"

### AARRR 检查
获取：小红书配色笔记 / 设计师社群
激活：第一次取色→1 秒出中式色名的"啊这有用"时刻
留存：历史记录 + 配色方案库，下次配色还来翻
推荐：生成海报带"小象取色"水印，设计师分享即拉新
收益：免费取色命名，付费高级配色 + 海报模板

## 使用方式

把想法告诉我，我来引导你完成这张卡片。你用自然语言说想法和目标用户，我按框架追问、提炼、检查虚荣指标。

**对话示例：**
- "我想做个帮设计师取色命名的工具" → 我引导你走三步流程，先问用户分层
- "我的北极星指标想定成 DAU" → 我帮你判断这是虚荣指标，给候选北极星
- "我不确定设计师愿不愿意为高级配色付费" → 我帮你定位这是最危险假设，设计手工 MVP 验证

## 与其他技能的关系

- **前置：** `xx-ai-or-not` 判断"该不该用 AI"通过后 → 用本技能定目标
- **并行：** `xx-research` 的调研结论是本技能"目标用户""核心问题"模块的素材
- **后续：** 目标定义卡 → `xx-business` 用精益画布展开商业模式
- **回溯：** `xx-track` 上线后用真实数据回校北极星指标和最危险假设是否成立
