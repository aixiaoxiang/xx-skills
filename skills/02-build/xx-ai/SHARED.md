# SHARED · AI 能力接入（平台无关）

> 本文件描述跨平台通用的 AI 能力接入原则。平台具体实现在 wechat/SKILL.md 和 web/SKILL.md。

**读者说明：** 本文件主要给 AI 跨平台执行约束（Prompt、降级、流式、成本任何平台都要遵守），次要给人判断标准（AI 接入是否健壮）。

## 通用原则

### 1. Prompt 工程
- 角色设定清晰
- 任务拆解明确
- 输出格式约束（JSON/Markdown）
- few-shot 示例

**AI 执行约束：** Prompt 必须显式约束输出格式，不能让模型自由发挥导致解析失败；few-shot 示例必须来自评估集（见 xx-data），不能凭空编。

**判断标准（给人）：** 同一输入跑 10 次，输出格式是否一致？不一致 = Prompt 约束不够。

### 2. 降级与容错
- 主备模型切换
- 超时重试
- 缓存策略
- 默认兜底

**AI 执行约束：** 主模型失败必须自动切备模型，备模型也失败必须返回兜底结果（不能给用户看报错页）。

### 3. 流式响应
- 打字机效果
- 错误中断处理
- 空结果处理

**AI 执行约束：** 流式中断必须保留已生成内容，不能整段清空；空结果要当 bad case 处理而非静默。

### 4. 成本控制
- 缓存命中
- 短 prompt
- 模型选择
- 限流策略

**判断标准（给人）：** 月度 AI 成本有没有上限？超了怎么办？没规划 = 迟早超预算。

**AI 执行约束：** 必须有超预算降级方案（限流/切便宜模型/付费墙），不能任由成本失控。
