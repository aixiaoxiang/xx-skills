# SHARED · 设计系统（平台无关）

> 本文件描述跨平台通用的设计系统原则。平台具体实现在 wechat/SKILL.md 和 web/SKILL.md。

**读者说明：** 本文件主要给 AI 跨平台执行约束（设计 Token 分层、对比度、字体比例任何平台都要遵守），次要给人判断标准（设计是否统一）。

## 通用原则

### 1. 设计 Token 理论
- 颜色、字号、间距、圆角用变量定义
- 暗色模式从第一天就规划
- Token 分层：基础变量 → 语义变量 → 组件变量

**AI 执行约束：** 任何 UI 色值/字号/间距必须引用 Token，禁止在组件里硬编码（如 `color: #0071E3` 直接写死）。

**判断标准（给人）：** 全项目搜 `#` 开头的色值，应该只在 Token 定义文件里出现。组件文件里出现硬编码色值 = 不合格。

### 2. 色彩体系
- 主色、辅色、中性色、状态色（成功/警告/错误）
- 色彩对比度需满足无障碍标准（WCAG AA）
- HSL 而非 HEX 便于调亮度

**AI 执行约束：** 文字与背景对比度必须 ≥ 4.5:1（WCAG AA）；品牌色定义必须同时给 HEX 和 HSL。

### 3. 字体层级
- 标题/正文/辅助文字的字号比例（如 1.250 黄金比）
- 行高、字间距规范
- 中英文混排适配

**判断标准（给人）：** 页面上字号种类不超过 7 种，超过 = 层级没理清。
