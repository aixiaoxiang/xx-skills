# SHARED · 组件库（平台无关）

> 本文件描述跨平台通用的组件库设计原则。平台具体实现在 wechat/SKILL.md 和 web/SKILL.md。

**读者说明：** 本文件主要给 AI 跨平台执行约束（组件设计原则任何平台都要遵守），次要给人判断标准（组件是否可复用）。

## 通用原则

### 1. 组件设计原则
- 单一职责：一个组件只做一件事
- 可组合：组件之间能自由搭配
- 可配置：通过 props/attributes 配置
- 可复用：跨页面跨项目使用

**AI 执行约束：** 一个组件不能同时做"取数据 + 渲染 + 业务逻辑"三件事；数据获取放页面，组件只负责渲染。

### 2. 组件分类
- 基础组件：按钮、输入框、卡片、列表
- 表单组件：上传、选择器、开关
- 反馈组件：弹窗、Toast、加载
- 布局组件：栅格、容器

### 3. 组件 API 设计
- 输入：props/attributes
- 输出：事件回调
- 状态：内部状态 vs 受控
- 默认值：所有可选 props 有合理默认值

**判断标准（给人）：** 拿到一个组件，不看文档能猜出 props 含义 = API 设计合格；猜不出 = 命名有问题。

**AI 执行约束：** 所有可选 props 必须有默认值，不能要求调用方必传；事件命名用 `onXxx` 或 `bindxxx` 格式。
