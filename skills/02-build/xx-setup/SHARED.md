# SHARED · 基础设施（平台无关）

> 本文件描述跨平台通用的基础设施原则。平台具体实现在 wechat/SKILL.md 和 web/SKILL.md。

**读者说明：** 本文件主要给 AI 跨平台执行约束（任何平台都要遵守），次要给人判断标准（基础设施是否合格）。

## 通用原则

### 1. 项目骨架原则
- 配置与代码分离
- 环境变量管理（开发/测试/生产）
- 错误码统一规范
- 日志记录标准化

**AI 执行约束：** 环境相关变量（envId、appid、API key）必须集中到单一配置文件，禁止散落在多个文件硬编码。

**判断标准（给人）：** 换一个环境（开发→生产）只需要改一个配置文件 = 合格；需要改多处 = 不合格。

### 2. 合规思路（各平台具体法规不同）
- 用户数据收集前需授权
- 隐私政策必须可访问
- 数据存储需加密
- 删除用户数据需支持

**AI 执行约束：** 涉及用户隐私的 API 调用前必须先弹授权，授权未通过不得调用；不得在小程序启动时一次性申请所有授权。

### 3. 部署流程
- 本地开发 → 测试环境 → 灰度 → 全量
- 版本号管理
- 回滚机制

**判断标准（给人）：** 上线出问题能不能在 10 分钟内回滚到上一版？不能 → 部署流程不合格。
